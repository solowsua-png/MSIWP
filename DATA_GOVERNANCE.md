# Data Governance

## Data Classification Levels

1. **Public**: Information that can be freely shared with the public without any impact on privacy or security.
2. **Internal Use Only**: Information that is restricted to the organization and should not be shared externally.
3. **Confidential**: Sensitive information that could cause harm if disclosed and should be accessible only to authorized individuals.
4. **Restricted**: Highly sensitive information that requires strict access controls and handling procedures.

## Data Residency Requirements for PNG

Organizations must adhere to the following data residency requirements in Papua New Guinea:
- Data must be stored within the country for all government-related data.
- Personal data of citizens must not be transferred outside the country without explicit consent.

## Data Lifecycle Management

1. **Creation**: Data is created and captured in various systems.
2. **Storage**: Data is stored in secure environments with adequate backup and recovery mechanisms.
3. **Usage**: Data is used strictly for its intended purpose, following organizational policies.
4. **Sharing**: Data sharing is governed by established protocols and compliance regulations.
5. **Archival**: Data that is not actively used should be archived securely for retrieval if necessary.
6. **Destruction**: Data must be securely disposed of when no longer needed, following organizational guidelines.

## Compliance Frameworks

Organizations must comply with relevant frameworks including but not limited to:
- **ISO/IEC 27001**: Information security management systems.
- **GDPR**: General Data Protection Regulation, where applicable.
- **Local laws and regulations pertaining to data protection and privacy.**

## Data Sovereignty Policies

- Data sovereignty ensures that data is subject to the laws and regulations of the country in which it is collected and stored.
- Organizations must ensure compliance with local laws regarding data use and storage.
